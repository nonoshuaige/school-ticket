## 一、项目概述

### 1.1 项目背景

面向毕业晚会、歌手大赛等校内热门活动的票务开售场景，构建一套从前端售票、订单创建、支付关单、取消回补到退款补偿的订单履约闭环系统。v3.0 新增笔记社区（发笔记、点赞、关注）。v3.1 新增发布笔记与二级展平评论系统。v3.2 引入 Redis ZSET 游标分页与 RabbitMQ 异步消息，实现笔记排行（最热/最新/我的）与关注/粉丝列表高性能查询。

### 1.2 核心功能

| 模块 | 功能 |
|------|------|
| 用户 | 注册、登录（手机号+密码，Cookie+JWT）、退出、关注/取关、关注/粉丝列表（游标分页）、关注统计 |
| 活动 | 首页双列网格、热卖中/预热中分组（按时间窗口）、真实最低票价、分页 |
| 购票 | 选择活动 → 选择票类 → 确认下单（名额扣减）→ 支付（模拟） |
| 订单 | 创建→支付→取消→退款→核销，五态流转，后端分页+状态筛选 |
| 我的 | 个人信息、关注数/粉丝数、订单入口 |
| 笔记 | 双列小红书风格卡片、三种排行（最热/最新/我的）、游标分页加载更多、发布笔记、点赞/取消、关注/取关、笔记详情、二级展平评论（发评论/回复/删除） |
| 导航 | 底部三 Tab：购票、笔记、我的 |

## 三、系统架构

### 3.1 认证流程

1. 登录成功 → 后端将 JWT 写入 httpOnly Cookie（name=token）
2. 前端 Axios 自动附带 Cookie（withCredentials: true）
3. JWT Cookie 过滤器拦截请求 → 解析 Cookie 中的 token → 提取 userId → 写入 CurrentUserHolder
4. 业务层通过 CurrentUserHolder.getUserId() 获取当前用户
5. 请求结束 → CurrentUserHolder.remove() 清除 ThreadLocal

### 3.2 CurrentUserHolder

```java
public class CurrentUserHolder {
    private static final ThreadLocal<Long> USER_ID_HOLDER = new ThreadLocal<>();
    public static void setUserId(Long userId) { USER_ID_HOLDER.set(userId); }
    public static Long getUserId() { return USER_ID_HOLDER.get(); }
    public static void remove() { USER_ID_HOLDER.remove(); }
}
```

> 约定：禁止在 Controller 参数中传递 userId。统一从 CurrentUserHolder 获取。

### 3.3 笔记模块认证

笔记列表接口为公开访问（`permitAll()`），但点赞和关注操作需要登录。Controller 层使用 `tryGetUserId()` 和 `requireUserId()` 两个辅助方法区分：
- `tryGetUserId()`：静默获取，未登录返回 null，列表仍正常返回但不带当前用户状态
- `requireUserId()`：强制获取，未登录抛出 `BusinessException(401, "请先登录")`

### 3.4 Redis ZSET 排行架构

笔记列表三种排行均基于 Redis ZSET（有序集合）实现游标分页：

| ZSET Key | Score | 用途 |
|----------|-------|------|
| `note:latest` | create_time 的 epoch 毫秒 | 最新排行 |
| `note:hottest` | like_count | 最热排行 |
| `note:mine:{userId}` | create_time 的 epoch 毫秒 | 我的笔记 |

关注/粉丝列表同样使用 ZSET：

| ZSET Key | Score | 用途 |
|----------|-------|------|
| `user:follow:{userId}` | System.currentTimeMillis() | 我关注的人 |
| `user:fans:{userId}` | System.currentTimeMillis() | 我的粉丝 |

**游标分页流程**：首次请求 cursor=null → `ZREVRANGEBYSCORE key +inf 0 LIMIT 0 pageSize` → 返回 lastScore 作为 nextCursor → 下次请求 cursor=nextCursor → `ZREVRANGEBYSCORE key (cursor-1) 0 LIMIT 0 pageSize`（排除 cursor 值，防止重复）。

**双写策略**：MySQL 写入成功后同步更新 Redis ZSET，同时发送 RabbitMQ 消息作为异步兜底。

### 3.5 RabbitMQ 异步消息

Topic Exchange `school.ticket.exchange` 绑定 3 个队列：

| Queue | Routing Key | 用途 |
|-------|------------|------|
| `note.like.queue` | `note.like` | 点赞/取消后异步更新 Redis ZSET |
| `note.create.queue` | `note.create` | 发布笔记后异步更新 Redis ZSET |
| `user.follow.queue` | `user.follow` | 关注/取关后异步更新 Redis ZSET |

> `RedisSyncListener.java` 中 3 个 `@RabbitListener` 方法消费消息，执行与同步写入相同的 ZSET 操作。

### 3.6 冷启动同步

系统启动后需调用同步接口，将 MySQL 存量数据全量写入 Redis ZSET：

```
POST /api/v1/note/sync-to-redis        # 同步笔记排行
POST /api/v1/user/follow/sync-to-redis # 同步关注关系（需登录）
```


## 五、订单状态机与履约流程

### 5.1 状态流转

```
[*] → 待支付(0) → 已支付(1) → 已核销(4)
待支付(0) → 已取消(2)  [主动取消/超时关单]
已支付(1) → 已退款(3)  [申请退款]
```

### 5.2 关键规则

**下单（名额扣减）**
- 事务内：SELECT ... FOR UPDATE 锁定票档行 → 校验余量 → UPDATE remaining_quantity -= quantity → 插入订单（status=0, expire_time=NOW()+15min）

**支付**（模拟）
- 验证订单存在且 status=0 且未过期 → 置 status=1, paid_time=NOW()

**取消/超时关单（回补）**
- 主动取消：POST /api/v1/order/cancel
- 超时关单：@Scheduled(fixedRate=30s) 扫描 expire_time < NOW() AND status=0
- 回补：UPDATE remaining_quantity += quantity, status=2

**退款**
- POST /api/v1/order/refund → 回补名额 + status=3

### 6.6 响应格式

```json

{ "code": 4001, "msg": "余票不足", "data": null }
```

### 6.7 分页响应格式

**MyBatis-Plus 分页**（活动列表 / 订单列表 / 评论列表使用）：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "records": [...],
    "total": 15,
    "size": 8,
    "current": 1,
    "pages": 2
  }
}
```

**游标分页 CursorPage**（笔记列表 / 关注列表 / 粉丝列表使用）：

```json
{
  "code": 200,
  "msg": "success",
  "data": {
    "records": [...],
    "nextCursor": "1781970000000",
    "hasMore": true,
    "total": 200
  }
}
```

> 游标分页首次请求 cursor=null，后续请求传入上次返回的 nextCursor。hasMore 为 true 时前端显示"加载更多"按钮。total 用于展示总数。

### 8.1 页面功能要点

**EventList（活动列表）**

- 评论输入框（登录门控）

**Mine（我的）**

## 附 A：开发问题记录

| 问题 | 解决 |
|------|------|
| 下单 403 | JWT Cookie Filter 补上 SecurityContextHolder.setAuthentication() |
| MySQL 8.0.36 CDN 不可达 | 改用阿里云镜像 8.0.28 |
| JDBC charset=utf8mb4 报错 | 改为 characterEncoding=UTF-8 |
| 前端无限跳回首页 | Pinia storeToRefs() 解构获取响应式 ref |
| 订单列表未区分状态 | 加 Tab 筛选 + 后端分页+状态筛选 |
| 未开售可购买 | isOnSale 计算属性控制票档禁用 |
| mysqld 启动报 ibdata1 must be writable | 管理员权限运行 PowerShell，`Start-Process -Verb RunAs` |
| PowerShell `mysql < init.sql` 报 RedirectionNotSupported | `<` 是 PS 保留运算符，改用 `cmd /c` 包装 |
| init.sql 执行报 No database selected | CREATE DATABASE / USE 被注释，加 `-D school_ticket` |
| pom.xml 自引用依赖导致 Maven 构建失败 | 删除 `<dependency>com.schoolticket:school-ticket</dependency>` |
| 活动卡片价格硬编码 | Event entity 加 @TableField(exist=false) minPrice，Service 批量查 ticket_category 取 MIN(price) |
| 笔记 N+1 查询用户信息 | 使用 userMapper.selectBatchIds() 批量加载 |
| 笔记点赞 N+1 查询 | LambdaQueryWrapper.in(noteIds) 批量查 + 按 noteId 分组 |
| gen_data.js 覆盖测试账号 | user_id=1 显式插入 '13800138000' / '测试用户' |
| 关注列表/笔记/点赞未落库 | 建表 + entity + mapper + service + controller 全套后端 |
| 笔记排行榜切换无变化 | van-tabs v-model 绑定问题，改用自定义 span 按钮 + 显式 switchSort() |
| gen_data.js 生成未来日期 | 日期生成范围限制到 2026-06-21 之前 |
| 前端分页组件不适用于游标 | 替换 van-pagination 为"加载更多"按钮 + nextCursor 驱动 |
| Redis isLiked 永远为 false | RedisNoteRankingService 中 isLiked 检查从未写入的 Redis key，改为 MySQL 批量查询 note_like 表 |
| /user/follow/sync-to-redis 403 | 该接口在 `/api/v1/**` authenticated() 规则下，需带登录 Cookie |
