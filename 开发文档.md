## 一、项目概述

### 1.1 项目背景
面向毕业晚会、歌手大赛等校内热门活动的票务开售场景，构建一套从前端售票、订单创建、支付关单、取消回补到退款补偿的订单履约闭环系统。

### 1.2 核心功能

| 模块 | 功能 |
|------|------|
| 用户 | 注册、登录（手机号+密码，Cookie+JWT）、退出 |
| 活动 | 首页活动列表、活动详情（含多档票价）、开售时间校验 |
| 购票 | 选择活动 → 选择票类 → 确认下单（名额扣减）→ 支付（模拟） |
| 订单 | 创建→支付→取消→退款→核销，五态流转 |
| 我的票证 | 按 Tab 筛选（全部/待支付/已支付），分页（10条/页） |
| 菜单导航 | 「购票」与「我的」底部导航，「我的」内子菜单进入订单 |
| 开售控制 | 未开售活动票档灰色 + 禁止购买，仅可浏览介绍 |

## 三、系统架构

### 3.1 认证流程

1. 登录成功 → 后端将 JWT 写入 httpOnly Cookie（name=token）
2. 前端 Axios 自动附带 Cookie（withCredentials: true）
3. JWT Cookie 过滤器拦截请求 → 解析 Cookie 中的 token → 提取 userId → 写入 CurrentUserHolder
4. 业务层通过 CurrentUserHolder.getUserId() 获取当前用户
5. 请求结束 → CurrentUserHolder.remove() 清除 ThreadLocal

### 3.2 CurrentUserHolder

```java
public class CurrentUserHolder {
    private static final ThreadLocal<Long> USER_ID_HOLDER = new ThreadLocal<>();
    public static void setUserId(Long userId) { USER_ID_HOLDER.set(userId); }
    public static Long getUserId() { return USER_ID_HOLDER.get(); }
    public static void remove() { USER_ID_HOLDER.remove(); }
}
```

> 约定：禁止在 Controller 参数中传递 userId。统一从 CurrentUserHolder 获取。

**user 表**
```sql
CREATE TABLE user (
  user_id     BIGINT       AUTO_INCREMENT PRIMARY KEY,
  phone       VARCHAR(20)  NOT NULL UNIQUE,

  nickname    VARCHAR(50)  DEFAULT '',

```

**event 表**
```sql
CREATE TABLE event (
  event_id        BIGINT       AUTO_INCREMENT PRIMARY KEY,
  title           VARCHAR(100) NOT NULL,
  description     TEXT,
  organizer       VARCHAR(100) NOT NULL,
  venue           VARCHAR(100) NOT NULL,
  poster_url      VARCHAR(255),
  sale_start_time DATETIME     NOT NULL,
  sale_end_time   DATETIME     NOT NULL,
  event_start_time DATETIME    NOT NULL,
  event_end_time  DATETIME     NOT NULL,
  status          TINYINT      DEFAULT 0,
  create_time     DATETIME     DEFAULT CURRENT_TIMESTAMP,
  update_time     DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE
);
```

**ticket_category 表**
```sql
CREATE TABLE ticket_category (
  ticket_id          BIGINT        AUTO_INCREMENT PRIMARY KEY,
  event_id           BIGINT        NOT NULL,
  name               VARCHAR(50)   NOT NULL,
  price              DECIMAL(10,2) NOT NULL,
  total_quantity     INT           NOT NULL,
  remaining_quantity INT           NOT NULL,
  description        VARCHAR(255),
  create_time        DATETIME      DEFAULT CURRENT_TIMESTAMP,
  update_time        DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE,
  FOREIGN KEY (event_id) REFERENCES event(event_id)
);
```

**order 表**
```sql
CREATE TABLE \`order\` (
  id            BIGINT        AUTO_INCREMENT PRIMARY KEY,
  order_no      VARCHAR(64)   NOT NULL UNIQUE,
  user_id       BIGINT        NOT NULL,
  ticket_id     BIGINT        NOT NULL,
  quantity      INT           DEFAULT 1,
  total_price   DECIMAL(10,2) NOT NULL,
  status        TINYINT       DEFAULT 0,
  expire_time   DATETIME      NOT NULL,
  create_time   DATETIME      DEFAULT CURRENT_TIMESTAMP,
  paid_time     DATETIME,
  cancel_time   DATETIME,
  refund_time   DATETIME,
  use_time      DATETIME,
  update_time   DATETIME      DEFAULT CURRENT_TIMESTAMP ON UPDATE,
  INDEX idx_user_status (user_id, status),
  FOREIGN KEY (user_id) REFERENCES user(user_id),
  FOREIGN KEY (ticket_id) REFERENCES ticket_category(ticket_id)
);
```

## 六、订单状态机与履约流程

### 6.1 状态流转

```
[*] → 待支付(0) → 已支付(1) → 已核销(4)
待支付(0) → 已取消(2)  [主动取消/超时关单]
已支付(1) → 已退款(3)  [申请退款]
```

### 6.2 关键规则

**下单（名额扣减）**
- 事务内：SELECT ... FOR UPDATE 锁定票档行 → 校验余量 → UPDATE remaining_quantity -= quantity → 插入订单（status=0, expire_time=NOW()+15min）

**支付**（模拟）
- 验证订单存在且 status=0 且未过期 → 置 status=1, paid_time=NOW()

**取消/超时关单（回补）**
- 主动取消：POST /api/v1/order/cancel
- 超时关单：@Scheduled(fixedRate=30s) 扫描 expire_time < NOW() AND status=0
- 回补：UPDATE remaining_quantity += quantity, status=2

**退款**
- POST /api/v1/order/refund → 回补名额 + status=3

## 附：开发问题记录

|------|------|
