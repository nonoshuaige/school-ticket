## 时序详解

```mermaid
sequenceDiagram
    actor 用户
    participant 前端 as Vue 前端
    participant API as Controller
    participant Svc as OrderService
    participant Lua as Redis Lua
    participant Cache as Redis Cache
    participant Stream as Redis Stream
    participant Bridge as Stream→MQ Bridge
    participant MQ as RabbitMQ
    participant Consumer as OrderCreateConsumer
    participant DB as MySQL
    participant Delay as 延时队列 (16min)
    participant DLX as 死信交换机

    用户->>前端: 点击下单
    前端->>前端: loading 态
    前端->>API: POST /order/create {ticketId,quantity}
    API->>Svc: createOrder(userId, req)
    Svc->>Svc: Caffeine 售罄短路 + 单飞锁
    Svc->>Svc: 查票档/活动
    Svc->>Svc: Snowflake 订单号

    Svc->>Lua: executePurchase()
    activate Lua
    Lua->>Lua: SET NX 防重 (1s TTL)
    Lua->>Lua: GET soldout / stock / 活动限购
    Lua->>Lua: DECRBY stock
    Lua->>Lua: HINCRBY event:purchase
    Lua->>Stream: XADD stream:orders
    Lua-->>Svc: {0, 'ok'}
    deactivate Lua

    Svc->>Cache: SET order:{no} stub JSON EX 1800
    Svc->>Cache: LPUSH user:orders:{userId} orderNo
    Svc->>Cache: LTRIM 0 9
    Svc-->>API: stub Order
    API-->>前端: {"orderNo":"...","status":0}
    前端-->>用户: 「排队中，稍后查看」

    Note over 用户,前端: 用户稍后查看订单列表
    用户->>前端: 打开订单列表
    前端->>API: GET /order/list
    API->>Cache: LRANGE user:orders:{userId} 0 9
    Cache-->>API: [orderNo...]
    API->>Cache: 逐个 GET order:{no}
    Cache-->>API: 订单 JSON 列表
    API-->>前端: 订单列表（0 次 DB）

    Note over Bridge,Stream: 异步桥接 (阻塞队列, 毫秒级)
    Bridge->>Stream: Reader XREAD BLOCK
    Stream-->>Bridge: 新消息 → put 队列
    Bridge->>Bridge: Worker take() 消费
    Bridge->>MQ: publish order.create
    Bridge->>Stream: XACK

    Note over Bridge,Stream: PEL 兜底 (每 5s)
    Bridge->>Stream: XPENDING
    Stream-->>Bridge: 卡住 >5s 的消息
    Bridge->>Stream: XRANGE
    Bridge->>MQ: publish (兜底)
    Bridge->>Stream: XACK

    MQ->>Consumer: @RabbitListener handleOrderCreate()
    Consumer->>DB: INSERT order
    Consumer->>Cache: updateOrderCache(完整数据)
    MQ->>Delay: 投递延时消息 (16min TTL)

    Note over Delay,DLX: 16 分钟后
    Delay-->>DLX: 消息过期 → 死信路由
    DLX->>Consumer: expireOrder()
    Consumer->>DB: UPDATE status=2
    Consumer->>DB: INSERT order_event_log
    Consumer->>Cache: refreshOrderCache()

    Note over 用户,API: 取消/退款 逆向链路
    用户->>API: POST /order/cancel (或 refund)
    API->>Svc: cancelOrder() / refundOrder()
    Svc->>DB: UPDATE status + 回补
    Svc->>DB: INSERT order_event_log (或 refund)
    Svc->>Cache: refreshOrderCache()

    Note over DB,Cache: 定时任务 每 10s
    loop OrderEventLogTask / RefundTask
        DB-->>Svc: 扫描待处理记录
        Svc->>Lua: rollback.lua (INCRBY + DEL soldout)
    end
```

## Redis Key 索引

| Key | 类型 | TTL | 说明 |
|-----|------|-----|------|
| `dedup:order:{userId}:{ticketId}` | String | 1s | 防重提交键，Lua SET NX |
| `ticket:stock:{id}` | String | saleEndTime | 票档库存，Lua DECRBY |
| `ticket:soldout:{id}` | String | 5min | 售罄标记 |
| `event:purchase:{id}` | Hash | — | 活动限购，userId→qty |
| `stream:orders` | Stream | MAXLEN~10000 | 订单事件流 |
| `order:{orderNo}` | String | 30min | 订单 JSON 缓存 |
| `user:orders:{userId}` | List | 30min | 最近 10 条订单号 |
